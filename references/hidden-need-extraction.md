# 隐性需求提炼方法

微醺 Skill 的任务不是把用户说的话改写得更漂亮，而是从原话里提炼“他真正被什么点燃”。

## 一、提炼对象

每次提炼时，优先找六类信号：

1. **高能原话**：用户突然变具体、激动、犀利、有画面感的话。
2. **情绪峰值**：兴奋、厌烦、愤怒、羡慕、不甘心、害怕、羞耻。
3. **审美线索**：音乐、影像、质感、颜色、速度、空间、人物原型。
4. **反复主题**：用户多次绕回来的词、场景、欲望或抱怨。
5. **反模式**：用户强烈拒绝、不想成为、不想做的东西。
6. **未完成冲动**：“我其实想……但……”背后的动作。

## 二、提炼表

对每个高能片段，用下表拆解：

| 字段 | 说明 |
|---|---|
| 原话 | 用户原句，不美化 |
| 表面想法 | 这句话字面上在说什么 |
| 情绪能量 | 这句话带着什么情绪 |
| 隐藏需求 | 用户真正想获得什么 |
| 审美 DNA | 声音、画面、节奏、质感 |
| 反模式 | 他拒绝什么 |
| 行动方向 | 可以转成什么实验 |

## 三、底色假设评分

一个“底色假设”必须通过三项评分。每项 0-3 分。

### 1. 能量密度

- 0 分：用户只是随口一说。
- 1 分：有一点情绪，但不具体。
- 2 分：有具体词、场景或比喻。
- 3 分：用户明显兴奋、愤怒、变快、变具体，出现画面和动作。

### 2. 复现性

- 0 分：只出现一次。
- 1 分：出现一次，但和其他回答有弱关联。
- 2 分：在不同问题中重复出现。
- 3 分：贯穿多个回答，且有不同表达形式。

### 3. 行动指向

- 0 分：无法转成动作。
- 1 分：只能转成很大的愿望。
- 2 分：可以转成小项目或内容方向。
- 3 分：可以在 7 天内做一个低成本实验。

总分 7 分以上，才可以作为强底色线索。4-6 分作为待验证线索。3 分以下只作为情绪碎片保留。

## 四、从抱怨到需求

抱怨不是需求，但抱怨常常保护着需求。

示例：

| 抱怨 | 可能的隐藏需求 |
|---|---|
| 这些工具都太像流程表 | 想要有生命感、有态度、有审美的工具 |
| 技术文章都像说明书 | 想要更有叙事、更有人味的技术表达 |
| 公司流程把人磨平了 | 想保留个人锋利度和创造空间 |
| 所有人都在做效率 | 想做一个不只是快，而是让人有感觉的东西 |

## 五、从审美到产品/内容方向

审美词不能只停留在形容词，要继续转成设计原则。

示例：

| 审美词 | 可能的设计原则 |
|---|---|
| 摇滚 | 保留粗粝感，允许锋利表达，不做过度圆滑 |
| 地下 | 小众但高辨识度，不讨好所有人 |
| 工业 | 结构清晰、硬朗、低装饰、高功能感 |
| 手作 | 有痕迹、有温度，不追求完美抛光 |
| 赛博 | 高密度信息、冷感界面、未来感符号 |
| 旷野 | 低束缚、大空间、非标准路径 |

## 六、诚实表达方式

输出时不要说：

- “你其实就是……”
- “你的本质是……”
- “这说明你的人格是……”

应该说：

- “一个可能的底色假设是……”
- “从这些原话看，比较强的线索是……”
- “这更像是一个待验证方向，而不是结论。”
- “如果你认可，我们可以把它压成一个 7 天实验。”
