# 示例：摇滚程序员

## 用户输入

> 我是程序员，但我一直觉得自己有点被工作压住了。我喜欢摇滚，喜欢那种现场感和失真感，但我做出来的东西总是很像普通效率工具。我想做一个开发者工具，但不想它又变成一个干净、正确、无聊的东西。

## 微醺 Skill 引导

先别把它当开发者工具。假设它是一支乐队，它是什么声音？

## 用户可能回答

> 它不是那种精致录音棚作品，更像一个地下乐队在小场地里演出。声音有点脏，但很真。它不应该装作高端 SaaS，也不应该到处说提升效率。我烦那些工具明明很无聊，还把自己包装成改变世界。

## 蒸馏结果节选

### 高能原话

1. “声音有点脏，但很真。”
2. “不应该装作高端 SaaS。”
3. “我烦那些工具明明很无聊，还把自己包装成改变世界。”

### 底色假设

你想做的可能不是“开发者效率工具”，而是一个**带现场感和反套路气质的开发者作品**。

你真正迷恋的不是摇滚这个标签，而是：

- 粗粝但真实
- 不过度包装
- 有现场感
- 不讨好所有人
- 让开发者感觉自己还活着，而不是只是工单机器

### 反模式

这个工具最不应该变成：

- 企业 SaaS 风格的效率平台
- 到处写“赋能开发者”的普通产品
- 过度 polished、没有性格的界面
- 把开发者当流程节点的东西

### 7 天实验

实验名称：**地下室开发者工具宣言**

最小动作：

做一个 GitHub README + 一张概念界面图，只表达这个工具的气质，不先做完整功能。

完成标准：

- 至少 3 个开发者看完后说“这个有点不一样”。
- 有人能说出它和普通效率工具的区别。
- 你自己看完仍然觉得它没有被驯化。

不做什么：

- 不做完整产品。
- 不写“提升效率 30%”。
- 不做企业官网风。
